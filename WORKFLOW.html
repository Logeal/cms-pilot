🚀 Workflow — Mise en ligne d'un site CMS

De l'achat du domaine aux 50 premiers articles publiés — étape par étape

1
Achat du nom de domaine Hostinger

~2 min · Coût : 8–12€/an

🌐
Acheter le domaine sur Hostinger
Aller sur hostinger.fr → rechercher le domaine → ajouter au panier → payer.
Important : ne pas prendre l'hébergement Hostinger, seulement le nom de domaine.
📋
Noter les nameservers actuels
Dans Hostinger → Domaines → DNS. Tu vas devoir les changer dans l'étape 3. Laisse pour l'instant.
2
Créer le VPS & installer Coolify Serveur

~15 min · Coût : ~8–15€/mois pour 50–100 sites

🖥️
Commander un VPS
Choisir Hostinger KVM 2 (~8€/mois) ou Contabo VPS S (~7€/mois).
OS : Ubuntu 22.04 LTS. Retenir l'IP du serveur (ex: 12.34.56.78).
RAM : 8 Go min
CPU : 4 vCPU min
Stockage : 80 Go min
🔧
Se connecter en SSH et installer Coolify
Ouvrir un terminal et coller ces commandes : 
ssh root@VOTRE_IP_VPS

curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash
Attendre ~5 min. Coolify est accessible sur http://VOTRE_IP:8000
🔑
Créer le compte admin Coolify
Ouvrir http://VOTRE_IP:8000 dans le navigateur → créer un compte admin → suivre le wizard de première configuration.
3
Pointer le domaine vers le VPS DNS

~5 min de config + jusqu'à 24h de propagation DNS

📡
Ajouter les enregistrements DNS sur Hostinger
Hostinger → Domaines → DNS/Nameservers → Gérer les DNS.
Ajouter ces deux enregistrements : 
Type  Nom    Valeur
A     @      VOTRE_IP_VPS
A     www    VOTRE_IP_VPS
⏳
Attendre la propagation DNS
En général 5–30 min, parfois jusqu'à 24h. Vérifier sur dnschecker.org que ton domaine pointe bien vers l'IP du VPS avant de continuer.
4
Déployer le CMS sur Coolify Déploiement

~10 min

📦
Mettre le projet CMS sur GitHub
Sur ton Mac, dans le dossier du CMS : 
git add .
git commit -m "Initial CMS"
git remote add origin https://github.com/TON_USER/cms-pilot.git
git push -u origin main
🚀
Créer une nouvelle application dans Coolify
Dans Coolify → New Resource → Application → GitHub → sélectionner le repo cms-pilot.

Configurer :
  • Build Pack : Nixpacks
  • Port : 3000
  • Domaine : tondomaine.fr
  • Activer HTTPS (SSL Let's Encrypt automatique)
⚙️
Renseigner les variables d'environnement
Dans Coolify → l'app → onglet Environment Variables, ajouter : 
DATABASE_URL=file:./prisma/dev.db
ADMIN_EMAIL=ton@email.com
ADMIN_PASSWORD=MotDePasseFort!
AUTH_SECRET=une_chaine_aleatoire_32_caracteres
NEXT_PUBLIC_APP_URL=https://tondomaine.fr
ANTHROPIC_API_KEY=sk-ant-...
UNSPLASH_ACCESS_KEY=ta_cle_unsplash
▶️
Lancer le déploiement
Cliquer Deploy. Coolify build l'app Next.js et la démarre automatiquement.
Le CMS est accessible sur https://tondomaine.fr
🔄
Initialiser la base de données
Dans Coolify → l'app → onglet Terminal (ou SSH sur le VPS dans le conteneur), exécuter : 
npx prisma migrate deploy
5
Configurer le site via le Setup Wizard CMS Admin

~5 min · Accessible sur https://tondomaine.fr/admin/setup

🔓
Se connecter à l'admin
Aller sur https://tondomaine.fr/login → entrer email + mot de passe définis dans les variables d'environnement.
🧙
Suivre le wizard de configuration (5 onglets)
  • Onglet 1 — Site : Nom du site (ex: "Tout sur le jardin")
  • Onglet 2 — Thème : Choisir parmi les 12 thèmes disponibles
  • Onglet 3 — Style : Palette de couleurs + typographie
  • Onglet 4 — Catégories : Ajouter les catégories du site
  • Onglet 5 — Accueil : Textes hero + images de catégories → la clé API est affichée ici
🗝️
Copier la clé API CMS
Dans l'onglet Accueil du wizard, la clé API du site est affichée. La copier maintenant — elle sera utilisée dans Pilot pour envoyer les articles.
La clé ressemble à : clxxxxxxxxxxxxxxxxxxxxx
6
Configurer Pilot pour ce site Pilot

~2 min

🔗
Lier le site Pilot au CMS
Dans Pilot → Sites → ouvrir le site correspondant → cliquer Modifier.
Descendre jusqu'à la section CMS et renseigner : 
URL du CMS  : https://tondomaine.fr
Clé API CMS : [la clé copiée à l'étape précédente]
Sauvegarder.
7
Générer et publier les 50 premiers articles Pilot

~30–60 min selon la vitesse de génération IA

✍️
Générer les articles depuis Pilot
Dans Pilot → Articles → Nouvel article.
Pour chaque article : sélectionner le site, la catégorie, le sujet, le mot-clé → Générer.
Faire 10 articles par catégorie (5 catégories × 10 = 50 articles).
Astuce : utiliser la génération en lot si disponible pour enchaîner les générations sans intervention.
📤
Envoyer chaque article au CMS
Pour chaque article généré dans Pilot :
Ouvrir l'article → vérifier le contenu → cliquer "Envoyer au CMS" (bouton en haut à droite).
Un lien vers l'article publié s'affiche en confirmation.
✅
Vérifier sur le site
Aller sur https://tondomaine.fr → vérifier que les articles apparaissent sur la homepage et dans les pages catégories.
8
Publication automatique des articles CMS

Cron actif — publication automatique toutes les heures

🕐
Cron de publication automatique
Le CMS contient un endpoint /api/admin/auto-publish déclenché toutes les heures par le cron configuré dans vercel.json.
Il publie automatiquement les articles dont la date de publication planifiée est passée.
Ce cron fonctionne nativement sur Vercel. Sur Coolify/VPS, il faut configurer un cron externe (ex: crontab ou Coolify Cron Jobs) qui appelle https://tondomaine.fr/api/admin/auto-publish toutes les heures avec le header x-api-key.
💾
Sauvegardes manuelles (automatique à venir)
La sauvegarde automatique vers Cloudflare R2 n'est pas encore implémentée.
En attendant, faire des sauvegardes manuelles via SSH : 
scp root@VOTRE_IP:/app/prisma/dev.db ./backup-$(date +%F).db
